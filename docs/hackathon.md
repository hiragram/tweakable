# RevenueCat Shipyard: Creator Contest まとめ

## 概要

インフルエンサー（クリエイター）と協力して、**4週間以内**に彼らのオーディエンス向けの**収益化可能なMVPモバイルアプリ**を作るコンテスト。

## 基本情報

| 項目 | 内容 |
|------|------|
| **締め切り** | 2026年2月12日 11:59pm PST（日本時間2月13日） |
| **賞金総額** | $140,000（現金） |
| **参加者数** | 5,475人（登録時点） |
| **参加条件** | 18歳以上、一部例外国を除く全世界から参加可能 |

## タイムライン

| 日程 | 内容 |
|------|------|
| 1月15日 | ブリーフ公開・登録開始 |
| 1月15日〜2月12日 | 開発期間（4週間） |
| 2月12日 | 提出締切（11:59pm PST） |
| 2月12日〜14日 | 審査パネル＆クリエイターによるレビュー |
| 2月26日 | 受賞者発表 |

## 必須要件

### 作るもの
- **プラットフォーム**: iOS（TestFlight）またはAndroid（Google Play Internal Testing）※最低1つ
- **収益化**: RevenueCat SDKを使ったサブスクまたはアプリ内課金の実装が必須
- **機能性**: コア機能が実装され、デモ可能な状態
- **オーディエンスフィット**: 選んだクリエイターのコミュニティの課題を解決するもの

### 提出物
1. **アプリアクセス** - TestFlightまたはGoogle Play Internal Testingリンク
2. **デモ動画（2〜3分）** - 主要機能のウォークスルー、オンボーディングから収益化までのユーザーフロー
3. **企画書（1〜2ページ）** - 課題設定、ソリューション概要、収益化戦略、今後のロードマップ
4. **技術ドキュメント** - アーキテクチャ概要、RevenueCat統合の詳細
5. **開発者プロフィール** - 経歴、ポートフォリオ、参加動機

## 賞金

各クリエイターのブリーフにつき **$20,000**（7ブリーフ × 1名 = 計7名受賞）

**受賞特典:**
- 賞金$20,000
- RevenueCat公式チャンネルでのフィーチャー
- クリエイターのチャンネルで紹介される機会
- ハッカソン後もクリエイターと協力してアプリを継続開発・収益化するチャンス

## 審査基準

| 基準 | 配点 | 内容 |
|------|------|------|
| **Audience Fit** | 30% | インフルエンサーのオーディエンスにどれだけマッチしているか |
| **User Experience** | 25% | 直感的で洗練された、使って楽しいか |
| **Monetization Potential** | 20% | サブスクモデルが魅力的で持続可能か |
| **Innovation** | 15% | ユニークな機能や新しいアプローチがあるか |
| **Technical Quality** | 10% | 安定していてパフォーマンスが良いか |

## 審査プロセス

1. 技術レビュー＆ブリーフ適合性の初期審査
2. 各ブリーフのトップ10〜15件をクリエイターがレビュー
3. クリエイターが最終的な受賞者を選出

## FAQ（重要ポイント）

- **提出数**: 最大7件（各クリエイターにつき1件まで）。ただし1つに集中することを推奨
- **App Store公開**: 提出前にストア公開は不要。ただし1月15日以降に開発を始めたものに限る
- **Webアプリ**: 不可。モバイルアプリ（iOS/Android）のみ
- **両OS対応**: 加点なし。片方でOK
